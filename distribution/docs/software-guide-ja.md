# フロントエンドキラー - ソフトウェアガイド

## 1. ソフトウェア概要

### 1.1 製品紹介

**フロントエンドキラー**は、ユーザーがさまざまなエミュレータゲームのデータ情報を簡単に管理、インポート、エクスポート、整理するのを助けるための専門的なエミュレータゲームメタデータ管理プラットフォームです。このソフトウェアは、Pegasus、EmulationStation DE (ES-DE)、RetroBatなどの主流エミュレータフロントエンドを含む複数のフロントエンド形式のデータのインポートとエクスポートをサポートしています。

### 1.2 主な特徴

- 🕹️ **マルチフォーマット対応**：Pegasus、ES-DE、RetroBatなどの複数のフロントエンド形式のインポートとエクスポートをサポート
- 📁 **バッチ処理**：ゲームデータとメディアファイルのバッチインポートとエクスポートをサポート
- 🔄 **プラットフォームマージ**：複数のゲームプラットフォームを統一管理のためにマージすることをサポート
- 🌐 **マルチ言語インターフェース**：中国語、英語、日本語の3つの言語インターフェースをサポート
- 📊 **データ統計**：詳細なデータ統計とレポート機能を提供
- 🎨 **メディア管理**：ゲームのメディアファイル（表紙、スクリーンショット、動画など）を自動的に管理およびマッチング
- ⚙️ **柔軟な設定**：異なるフロントエンドの要件に適応するためのカスタムルール設定をサポート
- 💾 **データベース管理**：ユーザーがデータを直接表示および管理するための組み込みH2データベース

## 2. ページ構造の概要

### 2.1 ホームページ (index.html)

ホームページはユーザーがシステムに入るためのエントリーポイントであり、ゲームスタイルのサイバーパンクデザインを採用し、明確な機能ナビゲーションを提供しています。

**主な機能エントリー：**

- **インポートモジュール**：データインポート機能へのクイックアクセス
- **データ処理**：プラットフォーム管理とデータ統計にアクセス
- **データマージ**：プラットフォームマージツールにアクセス
- **エクスポートモジュール**：データエクスポート機能にアクセス
- **システム設定**：システム設定、H2データベースコンソール、タスク管理などにアクセス

**トップ機能：**

- 🌍 **言語セレクター**：中国語、英語、日本語の間でインターフェース言語を切り替えることをサポート
- 🔔 **通知ベル**：システムタスクメッセージと通知を表示

### 2.2 インポートページ (data-import.html)

インポート機能を使用すると、ユーザーは外部フロントエンドのデータファイルをシステムデータベースにインポートできます。

**コア機能：**

- **マルチフォーマットデータファイルインポート**
  - XML形式（ES-DEのgamelist.xmlなど）をサポート
  - テキスト形式（Pegasusのmetadata.pegasus.txtなど）をサポート
  - ファイルエンコーディングと形式の自動認識

- **インテリジェントなフィールドマッピング**
  - 外部フィールドとシステムフィールドの自動マッチング
  - 複数値フィールドの処理をサポート
  - 柔軟なフィールドマッピング設定

- **メディアファイルインポート**
  - ゲームのメディアファイルを自動的にスキャンおよびインポート
  - 複数のメディアタイプ（表紙、スクリーンショット、動画など）をサポート
  - インテリジェントなメディアファイルマッチングルール

- **インポートテンプレート管理**
  - 複数のインポートテンプレート（ES-DE、Pegasusなど）をプリセット
  - カスタムインポートテンプレートをサポート
  - テンプレートのオンライン編集と設定

**使用プロセス：**

1. インポートデータソース（ディレクトリまたはファイル）を選択
2. インポートテンプレートを選択
3. インポートパラメータを設定
4. インポートを実行して結果を表示

### 2.3 プラットフォーム管理ページ (platform-management.html)

プラットフォーム管理はシステムのコアデータ管理モジュールであり、プラットフォームのCRUD操作と詳細な統計データを提供します。

**コア機能：**

- **プラットフォーム情報管理**
  - ゲームプラットフォームの追加、編集、削除
  - プラットフォームの基本情報（名前、説明、アイコンなど）の設定
  - プラットフォーム固有のパラメータの設定

- **ゲームリスト表示**
  - プラットフォーム上のすべてのゲームリストを表示
  - ゲームの検索とフィルタリングをサポート
  - ゲームの詳細情報とメディアファイルを表示

- **データ統計パネル**
  - プラットフォームのゲーム総数の統計
  - メディアファイルのカバレッジ統計
  - データ整合性分析
  - 視覚的なデータ表示

- **ゲームデータ編集**
  - ゲームの基本情報の編集
  - ゲームのメタデータの更新
  - ゲームのメディアファイルの関連付けの管理

### 2.4 プラットフォーム詳細ページ (platform-details.html)

単一のプラットフォームの詳細情報とゲームコンテンツを表示します。

**主なコンテンツ：**

- プラットフォームの基本情報の表示
- プラットフォームの統計データ
- ゲームリストとクイック操作
- プラットフォームの設定オプション

### 2.5 ゲームリストページ (game-list.html)

プラットフォーム上のすべてのゲームのリストビューを表示します。

**機能の特徴：**

- ゲームリスト表示（カード/リストビュー切り替え）
- ゲームの検索とフィルタリング
- ゲームのクイック操作（編集、削除、エクスポートなど）
- バッチ選択と操作

### 2.6 プラットフォームマージページ (platform-merge.html)

複数のプラットフォームのデータを1つに統合するための強力なプラットフォームマージツールです。

**コア機能：**

- **マージソースの選択**
  - 2つ以上のソースプラットフォームを選択
  - 各プラットフォームのゲームデータをプレビュー
  - プラットフォームの統計情報を表示

- **マージ設定**
  - ターゲットプラットフォーム名を設定
  - マージ戦略を選択（スキップ/上書き/マージ）
  - 競合処理ルールを設定

- **競合検出と解決**
  - 重複ゲームを自動的に検出
  - 競合の詳細を表示
  - 競合解決オプションを提供

- **マージレポート**
  - 詳細なマージレポートを生成
  - マージプロセス中の操作を記録
  - マージのロールバックをサポート（必要な場合）

### 2.7 マージ競合ページ (merge-conflicts.html)

マージプロセス中に見つかった競合情報を表示します。

**競合の種類：**

- ゲームの重複競合
- フィールド値の競合
- メディアファイルの競合

**解決オプション：**

- ソースプラットフォームのデータを保持
- ターゲットプラットフォームのデータを使用
- 手動でデータをマージ
- 競合アイテムをスキップ

### 2.8 マージレポートページ (merge-reports.html)

履歴マージ操作の詳細レポートを表示します。

**レポートの内容：**

- マージ操作の概要
- 処理されたゲームの数
- 競合解決の記録
- マージタイムライン

### 2.9 マージレポート詳細ページ (merge-report-details.html)

単一のマージレポートの完全な詳細情報を表示します。

### 2.10 エクスポートページ (export.html)

システム内のゲームデータをさまざまなフロントエンド形式にエクスポートします。

**コア機能：**

- **マルチフロントエンド形式のサポート**

- **エクスポートコンテンツの選択**

- **エクスポートルールの設定**
  - カスタムエクスポートルール
  - パステンプレートの設定

- **リアルタイムエクスポートの進行状況**

  - エラープロンプトと再試行

**エクスポートプロセス：**

1. ソースプラットフォームを選択

## 3. 機能モジュールの詳細

### 3.1 データインポートモジュール

#### 3.1.1 サポートされる形式

| 形式の種類 | ファイル形式 | 適用可能なフロントエンド |
|----------|------------|-------------------|
| XML形式 | gamelist.xml | ES-DE, RetroBat |
| テキスト形式 | metadata.pegasus.txt | Pegasus |
| JSON形式 | カスタム | カスタムテンプレート |

#### 3.1.2 インポートプロセス

```
データソース選択 → テンプレートマッチング → フィールドマッピング → データ検証 → インポート実行 → 結果フィードバック
```

#### 3.1.3 メディアファイルの処理

- メディアディレクトリの自動スキャン
- インテリジェントなメディアファイルのマッチング
- 複数の画像および動画形式のサポート
- メディアファイルの重複排除と最適化

### 3.2 データ処理モジュール

#### 3.2.1 プラットフォーム管理

- プラットフォームのCRUD操作
- プラットフォーム設定の管理
- プラットフォームの統計情報

#### 3.2.2 ゲーム管理

- ゲーム情報の編集
- バッチ操作のサポート
- メディアファイルの関連付け
- ゲームデータの検証

#### 3.2.3 データ統計

- ゲームの総数の統計
- プラットフォームの比較分析
- メディアのカバレッジ分析
- データ整合性チェック

### 3.3 データマージモジュール

#### 3.3.1 マージ戦略

- **インテリジェントマージ**：重複データを自動的に処理
- **完全上書き**：古いデータを新しいデータで完全に置き換え
- **選択的マージ**：保持するデータを手動で選択
- **新規のみマージ**：存在しないゲームのみを追加

#### 3.3.2 競合処理

- 重複ゲームを自動的に検出
- 競合の詳細を表示
- 解決の提案を提供
- 手動解決をサポート

### 3.4 データエクスポートモジュール

#### 3.4.1 エクスポートオプション

- ✅ ゲームデータファイル
- ✅ ゲームROMファイル
- ✅ メディアファイル
- ✅ プラットフォーム設定ファイル

#### 3.4.2 パステンプレート変数

- `{platform}` - プラットフォーム名
- `{gameName}` - ゲーム名
- `{outputPath}` - エクスポートのルートパス
- `{mediaPath}` - メディアファイルパス
- `{romsPath}` - ROMファイルパス

#### 3.4.3 データファイル形式

各フロントエンドの要件を満たす標準形式のファイルを生成することをサポートし、XMLおよびプレーンテキスト形式を含みます。

## 4. 使用プロセスガイド

### 4.1 初回使用プロセス

1. **システムにアクセス**：ブラウザを開いてシステムアドレスにアクセス
2. **言語設定**：必要に応じてインターフェース言語を切り替え
3. **システムの設定**：システム設定でデータベースとパスを設定
4. **使用を開始**：データをインポートするか手動でプラットフォームデータを作成

### 4.2 日常使用プロセス

#### 新しいゲームデータのインポート

1. 「インポート」ページに入る
2. データソースディレクトリを選択
3. 適切なインポートテンプレートを選択
4. インポートパラメータを設定
5. インポートを実行して結果を表示

#### ゲームの管理と編集

1. 「データ処理」ページに入る
2. 管理するプラットフォームを選択
3. ゲームリストを表示
4. ゲーム情報を編集
5. 変更を保存

#### ゲームデータのエクスポート

1. 「エクスポート」ページに入る
2. ソースプラットフォームを選択
3. ターゲットフロントエンド形式を選択
4. エクスポートオプションを設定
5. エクスポートを実行

### 4.3 高度な操作プロセス

#### プラットフォームのマージ

1. 「データマージ」ページに入る
2. ソースプラットフォームを選択（複数選択可能）
3. マージオプションを設定
4. マージの競合を処理
5. マージを実行してレポートを表示

#### カスタムインポート/エクスポートテンプレート

1. テンプレート設定のドキュメントを参照
2. 新しいテンプレートJSONファイルを作成
3. フィールドマッピングルールを設定
4. メディアファイルのマッチングルールを設定
5. テンプレートを保存してテスト

## 5. 設定ファイルの説明

### 5.1 インポートテンプレート設定

インポートテンプレートは `/data/rules/import-templates/` ディレクトリに格納され、主に以下を含みます：

- **ES-DEテンプレート** (`esde.json`)：EmulationStation DE形式に適しています
- **Pegasusテンプレート** (`pegasus.json`)：Pegasusフロントエンド形式に適しています
- **汎用テンプレート** (`generic.json`)：汎用形式に適しています

### 5.2 エクスポートルール設定

エクスポートルールは `/data/rules/export-rules/` ディレクトリに格納され、主に以下を含みます：

- **esde.json**：ES-DEエクスポートルール
- **pegasus.json**：Pegasusエクスポートルール
- **retrobat.json**：RetroBatエクスポートルール
- **template.json**：汎用エクスポートテンプレート

### 5.3 翻訳設定

翻訳API設定は `/data/rules/translation-config.json` に格納され、機械翻訳サービスの設定に使用されます。

## 8. よくある質問

### 8.1 インポート関連

**Q: インポート時にファイル形式がサポートされていないと表示されたらどうしますか？**
A: ファイル形式がサポートされている形式（XML、テキストなど）であるか確認し、ファイルエンコーディングがUTF-8であることを確認してください。

**Q: メディアファイルが正しくマッチしない場合はどうしますか？**
A: インポートテンプレートのmediaRules設定を確認し、パステンプレートが実際のメディアファイル構造と一致していることを確認してください。

### 8.2 エクスポート関連

**Q: エクスポートされたファイルがフロントエンドの要件を満たしていない場合はどうしますか？**
A: エクスポートルール設定ファイルをカスタマイズして、フィールドマッピングとデータ形式を調整することができます。詳細については、エクスポート機能のドキュメントを参照してください。

**Q: メディアファイルがエクスポートされない場合はどうしますか？**
A: エクスポート設定のメディアルール設定を確認し、メディアファイルが存在し、パス設定が正しいことを確認してください。

### 8.3 システム関連

**Q: データベースをバックアップするにはどうすればよいですか？**
A: システム設定ページで「データベースのバックアップ」ボタンをクリックすると、システムはデータベースを `/data/backup` ディレクトリにバックアップします。

**Q: データベースを復元するにはどうすればよいですか？**
A: システム設定ページで復元するバックアップファイルを選択し、「データベースの復元」ボタンをクリックしてください。

**Q: H2データベースコンソールに接続できない場合はどうしますか？**
A: H2コンソールのJDBC URL設定が正しいか確認してください。デフォルトは `jdbc:h2:file:/data/database` です。
