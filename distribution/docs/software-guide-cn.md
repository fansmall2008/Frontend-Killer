# 前端杀手 - 软件说明文档

## 1. 软件简介

### 1.1 软件概述

**前端杀手**是一款专业的模拟游戏元数据管理平台，旨在帮助用户轻松管理、导入、导出和整理各种模拟器游戏的数据信息。该软件支持多种前端格式的数据导入和导出，包括Pegasus、EmulationStation DE (ES-DE)、RetroBat等主流模拟器前端。

### 1.2 主要特性

- 🕹️ **多格式支持**：支持导入导出Pegasus、ES-DE、RetroBat等多种前端格式
- 📁 **批量处理**：支持批量导入导出游戏数据和媒体文件
- 🔄 **平台合并**：支持将多个游戏平台合并为一个统一管理
- 🌐 **多语言界面**：支持中文、英文、日文三种语言界面
- 📊 **数据统计**：提供详细的数据统计和报告功能
- 🎨 **媒体管理**：自动管理和匹配游戏媒体文件（封面、截图、视频等）
- ⚙️ **灵活配置**：支持自定义规则配置，适应不同的前端需求
- 💾 **数据库管理**：内置H2数据库，方便用户直接查看和管理数据

## 2. 页面结构概览

### 2.1 首页 (index.html)

首页是用户进入系统的入口页面，采用游戏风格的赛博朋克设计，提供清晰的功能导航。

**主要功能入口：**

- **导入模块**：快速进入数据导入功能
- **数据处理**：进入平台管理和数据统计
- **数据合并**：进入平台合并工具
- **导出模块**：进入数据导出功能
- **系统设置**：访问系统配置、H2数据库控制台、任务管理等

**顶部功能：**

- 🌍 **语言选择器**：支持在中文、英文、日文之间切换界面语言
- 🔔 **通知铃铛**：显示系统任务消息和通知

### 2.2 导入页面 (data-import.html)

导入功能允许用户将外部前端的数据文件导入到系统数据库中。

**核心功能：**

- **多格式数据文件导入**
  - 支持XML格式（如ES-DE的gamelist.xml）
  - 支持文本格式（如Pegasus的metadata.pegasus.txt）
  - 自动识别文件编码和格式

- **智能字段映射**
  - 自动匹配外部字段与系统字段
  - 支持多值字段处理
  - 灵活的字段映射配置

- **媒体文件导入**
  - 自动扫描和导入游戏媒体文件
  - 支持多种媒体类型（封面、截图、视频等）
  - 智能媒体文件匹配规则

- **导入模板管理**
  - 预设多种导入模板（ES-DE、Pegasus等）
  - 支持自定义导入模板
  - 模板在线编辑和配置

**使用流程：**

1. 选择导入数据源（目录或文件）
2. 选择导入模板
3. 配置导入参数
4. 执行导入并查看结果

### 2.3 平台管理页面 (platform-management.html)

平台管理是系统的核心数据管理模块，提供平台的增删改查和详细的统计数据。

**核心功能：**

- **数据统计面板**

  - 管理游戏媒体文件关联

## 5. 配置文件说明

### 5.1 导入模板配置

导入模板存储在 `/data/rules/import-templates/` 目录，主要包括：

- **ES-DE模板** (`esde.json`)：适用于EmulationStation DE格式
- **Pegasus模板** (`pegasus.json`)：适用于Pegasus前端格式
- **通用模板** (`generic.json`)：适用于通用格式

### 5.2 导出规则配置

导出规则存储在 `/data/rules/export-rules/` 目录，主要包括：

- **esde.json**：ES-DE导出规则
- **pegasus.json**：Pegasus导出规则
- **retrobat.json**：RetroBat导出规则
- **template.json**：通用导出模板

### 5.3 翻译配置

翻译API配置存储在 `/data/rules/translation-config.json`，用于配置机器翻译服务。

## 8. 常见问题解答

### 8.1 导入相关

**Q: 导入时提示文件格式不支持怎么办？**
A: 请检查文件格式是否为支持的格式（XML、文本等），并确保文件编码为UTF-8。

**Q: 媒体文件没有正确匹配怎么办？**
A: 检查导入模板中的mediaRules配置，确保路径模板与实际媒体文件结构匹配。

### 8.2 导出相关

**Q: 导出的文件不符合前端要求怎么办？**
A: 可以自定义导出规则配置文件，调整字段映射和数据格式。具体请参考导出功能文档。

**Q: 媒体文件没有导出怎么办？**
A: 检查导出配置中的媒体规则设置，确保媒体文件存在且路径配置正确。

### 8.3 系统相关

**Q: 如何备份数据库？**
A: 在系统设置页面点击"备份数据库"按钮，系统会将数据库备份到 `/data/backup` 目录。

**Q: 如何恢复数据库？**
A: 在系统设置页面选择要恢复的备份文件，点击"恢复数据库"按钮。

**Q: H2数据库控制台无法连接怎么办？**
A: 检查H2控制台的JDBC URL配置是否正确，默认应为 `jdbc:h2:file:/data/database`。

## 11. Docker 部署指南

### 11.1 使用 docker-compose 部署（推荐）

```bash
# 创建数据目录
mkdir -p logs roms output rules input backup database

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 11.2 使用 docker run 部署

```bash
# 创建数据目录
mkdir -p logs roms output rules input backup database

# 运行容器
docker run -d \
  --name webgamelistoper \
  -p 8080:8080 \
  -v $(pwd)/logs:/data/logs \
  -v /path/to/roms:/data/roms \
  -v $(pwd)/output:/data/output \
  -v $(pwd)/rules:/data/rules \
  -v $(pwd)/backup:/data/backup \
  -v $(pwd)/database:/data/database \
  -e SPRING_PROFILES_ACTIVE=default \
  -e SERVER_TOMCAT_BASEDIR=/data \
  -e JAVA_OPTS="-Xmx2g -Xms512m -XX:+UseG1GC" \
  -e SPRING_RESOURCES_STATIC_LOCATIONS=classpath:/static/,file:/data,file:/data/roms,file:/data/output,file:/data/input \
  --restart unless-stopped \
  fansmall/webgamelistoper:1.0.2-beta
```

### 11.3 常用 Docker 命令
